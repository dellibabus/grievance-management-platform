Table Reference

TablePurpose
rolesDynamic roles with a permissions jsonb array. Super admins can create/edit/delete roles via the UI.
permissionsMaster catalog of permission names & descriptions, referenced by roles.
usersAll staff accounts. district_id is null for state-level roles (super_admin / state_admin).
districts / mandals / villagesGeographic hierarchy (cascade delete downward) used for scoping complaints and users.
categoriesComplaint categories shown in dropdowns (Infrastructure, Healthcare, etc.)
complaintsCore grievance ticket — status, priority, location, category, assignee, creator.
complaint_updatesAppend-only timeline of status changes and comments.
attachmentsUploaded files (image/pdf/video) linked to a complaint.
assignmentsHistory log of every assignment action.
notificationsPer-user in-app notification feed.
refresh_tokensServer-side store of 7-day refresh tokens for revocation/logout.
audit_logsSystem-wide audit trail of sensitive actions (login, CRUD on users/complaints/roles).
